Database Security

Introduction

Database security refers to the processes, tools, policies, and techniques used to protect a database from unauthorized access, misuse, corruption, loss, or damage. It ensures that data stored in a database remains confidential, accurate, and available to authorized users only. In simple terms, database security protects data from threats and ensures it remains safe, correct, and accessible when needed.


Importance of Database Security

i. Protection of Sensitive Information - Databases often store personal data, financial records, health information, and business secrets. Security helps prevent theft or exposure.

ii. Preventing Unauthorized Access - Strong security measures ensure only the right people can view or modify information.

iii. Maintaining Data Integrity - Security protects data from being altered, deleted, or corrupted by accident or malicious attacks.

iv. Ensuring Business Continuity - Organizations rely on databases to operate. Security ensures the database remains available, even during failures.

v. Compliance with Laws and Regulations - Many countries and industries have data protection laws (e.g., GDPR, HIPAA). Database security ensures compliance.

vi. Preventing Financial Loss and Reputation Damage - Data breaches can cause loss of money, customer trust, and brand reputation.


Access Control in Database Security

Access control refers to the rules and mechanisms that determine who can access the database and what actions they are allowed to perform.


Types of Access Control:

i. Discretionary Access Control (DAC) - The owner of the data decides who gets access. Example: A manager granting access to certain staff.

ii. Mandatory Access Control (MAC) - Access is based on predefined security levels (e.g., Top Secret, Confidential). Users must have the appropriate clearance.

iii. Role-Based Access Control (RBAC) - Access is based on user roles within an organization.

Example:

i. A cashier can enter sales data

ii. A supervisor can update sales records

iii. An admin can manage all users


Encryption

Encryption is the process of converting data into a coded form so that it cannot be understood by unauthorized users.


Types of Encryption in Databases:

i. Data-at-Rest Encryption - Secures stored data on disks or backup files.

ii. Data-in-Transit Encryption - Protects data moving between systems, such as between a web app and a database server.

iii. Column-level or Field-level Encryption - Specific sensitive data such as passwords, credit card numbers, or medical data are encrypted.


Importance of Encryption

i. Prevents data theft even when hackers break into the server

ii. Protects sensitive information

iii. Helps organizations meet compliance requirements

iv. Ensures data privacy and confidentiality

Data Integrity

Integrity refers to the correctness, accuracy, and consistency of data in a database.


Ways to Maintain Data Integrity:

i. Validation rules (e.g., age cannot be negative)

ii. Constraints (primary key, foreign key, unique, check constraints)

iii. Backups and transaction logs

iv. Avoiding duplicate records

v. Preventing unauthorized changes


Types of Integrity:

Entity Integrity: Each record must have a unique primary key

Referential Integrity: Relationship between tables must remain consistent

Domain Integrity: Data values must be valid according to defined rules


Data Availability

Availability ensures that authorized users can access the database whenever needed.


How Availability is Ensured:

i. Regular backups

ii. Redundant systems and failover servers

iii. Reliable power supply and network connections

iv. Database performance tuning

v. Protection against Denial-of-Service (DoS) attacks

vi. Disaster recovery plans

High availability ensures continuous access, especially for systems that run 24/7 (banks, hospitals, airlines, etc.).


Role of the Database Administrator (DBA) in Security

The Database Administrator plays a central role in maintaining database security.


Key Responsibilities of the DBA:

i. User Management

- Creating and deleting user accounts

- Assigning roles and privileges

- Enforcing strong authentication methods


ii. Implementing Access Control Policies

Ensuring that users only access the data they are authorized to use.


iii. Monitoring and Auditing Database Activities

Tracking user actions through logs to detect suspicious activities early.


iv. Backup and Recovery Management

Making regular backups and designing disaster recovery strategies.


v. Applying Security Patches and Updates

Keeping the database software updated to protect against vulnerabilities.


vi. Designing and Enforcing Security Policies

Setting rules for password strength, data access, and system usage.


Summary

Database security ensures the confidentiality, integrity, and availability of data. It involves access control, encryption, integrity checks, and availability measures. The Database Administrator plays a crucial role in implementing, managing, and monitoring all security practices to protect the database fromthreats.

Computer Software Maintenance (Contd.)

Software maintenance refers to the process of modifying, updating, and improving software applications after they have been delivered and deployed. It ensures that the software continues to function correctly, efficiently, and securely over time.

It is one of the longest phases in the Software Development Life Cycle (SDLC) because software must adapt to changing technologies, user needs, and security requirements.

Types of Software Maintenance


a. Corrective Maintenance

This type of maintenance involves fixing bugs, errors, and defects found in the software after it has been released. Example is fixing a crash issue in a mobile app or correcting an incorrect calculation in an accounting software.


b. Adaptive Maintenance

This is carried out when software needs to be updated to work with new environments or technologies. Example is updating software to work with a new operating system version, database, or web browser.


c. Perfective Maintenance

This involves improving or enhancing software performance and features without changing its core functions. Example include adding new features to a website, improving speed, or enhancing the user interface.

d. Preventive Maintenance

This is carried out to prevent future software issues. It includes code optimization, documentation updates, and security patching. Example is rewriting parts of the code to make it more efficient or reducing memory leaks.


Importance of Software Maintenance

i. Fixes existing problems and improves performance.

ii. Adapts software to new environments.

iii. Extends the life span of software.

iv. Ensures data and system security.

v. Enhances user satisfaction.


Common Software Problems and Solutions

Problem Possible Cause Solution
Application crashes Bug in the code or corrupt file Apply patch or reinstall software
Slow performance Memory leaks, too many background tasks Optimize code and close unused processes
Software not installing Incompatibility with OS Update OS or use compatible version
Security breach Outdated software or weak encryption Apply security patches, use firewalls
License error Invalid or expired license key Renew license or contact vendor

Safety and Ethical Considerations in Software Use

i. Avoid using pirated or unlicensed software.

ii. Respect copyright and intellectual property laws.

iii. Regularly update antivirus and security software.

iv. Backup important files before major updates.

v. Follow organizational IT policies and best practices.


Difference Between Hardware and Software

Feature Hardware Software
Tangibility Tangible (can be touched) Intangible (cannot be touched)
Function Performs physical operations Controls and instructs hardware
Durability Wears out over time Does not wear out but may become obsolete
Replacement Must be replaced physically Can be updated or reinstalled

Conclusion:

Computer software is the soul of a computer system — without it, hardware would be useless. It drives all computer operations, from system management to user applications.

To keep software reliable, efficient, and secure, proper software maintenance is essential. Regular updates, error fixes, and user-centered improvements ensure that software continues to meet the ever-changing needs of technology and society.

Crash Recovery

Introduction

In database systems, crash recovery refers to the process of restoring the database to a consistent state after a failure. Databases are expected to provide reliability and durability, ensuring that once a transaction is committed, its effects are permanent — even in the event of system crashes, power failures, or software errors.


Failures can occur due to:

i. System crashes (hardware/software failure)

ii. Transaction failures

iii. Media failures (disk corruption)

iv. Power outages

To ensure data durability, database systems use recovery mechanisms that rely heavily on logs and checkpointing.


Objectives of Crash Recovery

i. Preserve atomicity – either all actions of a transaction are completed or none are.

ii. Maintain consistency – the database remains in a valid state before and after recovery.

iii. Ensure durability – committed transactions must survive crashes.

iv. Minimize downtime – restore the database quickly to operational state.


Types of Failures in Databases


1. Transaction Failure

- Occurs when a transaction cannot complete successfully (e.g., logical errors, deadlocks).

- Requires rollback of changes made by that transaction.

2. System Failure

- System crash causes all active transactions to be lost from memory

3. Media Failure

- Permanent loss or corruption of data (e.g., disk crash), requiring data restoration from backups.


ARIES (Algorithm for Recovery and Isolation Exploiting Semantics)

ARIES is one of the most widely used and advanced recovery algorithms, adopted by major database systems such as IBM DB2, SQL Server, and Oracle.

It is based on three main principles:


i. Write-Ahead Logging (WAL) - All changes must be logged before being written to disk.


ii. Repeating History During Redo - Redo all actions from the log to reconstruct the state of the database before failure.


iii. Logging Changes During Undo - While undoing, ARIES logs compensation records to ensure that the undo actions can also be recovered if another crash happens during recovery.


Importance of Crash Recovery

i. Data durability — committed data is never lost.

ii. System reliability - database can recover automatically after failure.

iii. Business continuity - minimizes downtime and operational impact.

iv. Data integrity - prevents inconsistent or partial updates.


Conclusion:

Crash recovery is an essential component of modern database systems, ensuring atomicity, consistency, and durability.

Algorithms like ARIES make this process efficient, reliable, and recoverable even from repeated failures. By maintaining proper logs, checkpoints, and recovery data structures, database systems can restore operations quickly and preserve the integrity of stored data.
